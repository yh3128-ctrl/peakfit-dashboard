# 100대명산 마스터 데이터 설명서

## 1. 파일 개요

| 항목 | 내용 |
|------|------|
| **파일명** | `100mountains_master.csv` |
| **위치** | `peakfit/data/100mountains_master.csv` |
| **용도** | 100대명산 등산 추천 서비스의 핵심 데이터 |
| **구조** | 319행 × 44컬럼 |
| **단위** | **코스 단위** (같은 산이라도 등산 코스마다 1행) |
| **산 수** | 100개 산 (코스별로 펼쳐져 319행) |
| **인코딩** | UTF-8 (BOM 포함, 엑셀 호환) |

이 파일은 산림청 공공데이터 API, 숲나들e 100대명산 정보, 계절 추천 데이터, 국립공원 계단 시설 정보, 재난위험지구 데이터 등 **총 9종의 원본 데이터**를 하나로 통합하여 만든 마스터 데이터입니다. 사용자가 난이도, 소요시간, 계절, 접근성 등 다양한 조건을 입력하면 그에 맞는 산과 코스를 추천해주는 서비스의 기반 데이터로 활용됩니다.

## 4. 데이터 통합 과정

### 4-1. 조인키(Join Key) 전략

```
숲나들e (명산_이름) ──┐
                     ├──→ [산이름 매핑] ──→ 마스터
seasonal (산이름)  ───┘        ↕
                        frtrlNm 매칭
                              ↕
peak_poi (frtrlNm) ───┐
sghtng_poi (frtrlNm) ─┤
trnsp_poi (frtrlNm) ──┼──→ [frtrlNm 기준 집계] ──→ 마스터
entrance (frtrlNm) ───┤
scenery (frtrlNm) ────┘
                              ↕
                        산이름 텍스트 매칭
                              ↕
계단 시설 (상대위치) ──┐
                      ├──→ [텍스트 매칭 집계] ──→ 마스터
재난위험 (지구명) ─────┘
```

- **산림청 API 데이터** 5종은 `frtrlId`와 `frtrlNm`(산 이름)을 공통 키로 공유합니다.
- `frtrlId ↔ frtrlNm`은 **1:1 관계** (96쌍)
- 숲나들e의 `명산_이름`과 API의 `frtrlNm`은 대부분 일치하지만, **16건의 이름 불일치**가 있어 수동 매핑 딕셔너리로 해소했습니다.
  - 예: `가야산(경남)` → `가야산`, `백운산` → `백운산(광양)`, `내변산` → `변산`

### 4-2. 파생 점수 산출 기준

**난이도_점수 (1.0 ~ 5.0)**
- 높이 기반: 500m 미만=1, ~800m=2, ~1200m=3, ~1500m=4, 1500m+=5
- 난이도등급 기반: 초급=1, 중급=3, 고급=5
- 소요시간 기반: 2시간 미만=1, ~3시간=2, ~4시간=3, ~5시간=4, 5시간+=5
- **계단 시설 기반**: 5개 미만=1, ~10개=2, ~15개=3, ~20개=4, 20개+=5 (총 길이 300m 이상이면 +1 보정)
- → 네 지표의 **평균** (계단 데이터가 없는 산은 나머지 지표들의 평균)

**접근성_점수 (1.0 ~ 5.0)**
- 교통수단 수, 대중교통접근(O/X), 케이블카(O/X), 포장길 비율을 종합한 **평균**

**볼거리_점수 (1 ~ 5)**
- 조망점 수 + 경관포인트 수 + 문화자원 수의 합계를 구간별 등급 부여

**추천_등급**
- `입문자추천=O` → 초보자 추천
- `난이도_점수 ≤ 2` → 초보자 적합
- `난이도_점수 ≤ 3` → 중급자 적합
- `난이도_점수 > 3` → 상급자 적합
